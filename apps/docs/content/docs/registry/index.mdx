Campfire distributes layouts and components through a **shadcn-compatible
registry** — no custom format, no hidden runtime dependencies. Items are
copied as source straight into your repo:

```txt
components/   registry components install here
layouts/      registry layouts install here
```

There is no special registry folder — installed files sit next to your own
and are yours to edit. If an installed file would collide with one of yours,
review the diff shadcn shows before accepting the overwrite.

## Installing

```bash
camp add metric-card
camp add problem-solution
```

Bare names resolve to this site: `/r/<name>.json`. Anything
shadcn-compatible works too:

```bash
bunx shadcn@latest add https://campfire.dev/r/metric-card.json
```

## Principles

- The registry distributes **source code into your repo**.
- Installation behavior is exactly shadcn's — `camp add` only resolves nicer
  names.
- Item metadata uses shadcn's schema: `files[].target` places code under
  `layouts/` and `components/`; `registryDependencies` pulls in supporting
  primitives.

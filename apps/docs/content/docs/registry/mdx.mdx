Most of a deck isn't components — it's markdown. `components/mdx.tsx` is the
file that decides how that markdown renders at slide scale, and it ships
through the registry like everything else.

<ComponentPreview name="mdx">

```mdx title="slides/02-markdown.mdx"
## Markdown, at slide scale

Plain prose with `inline code` and [links](https://campfire.dev) renders
through `components/mdx.tsx`.

- Slides are MDX files
- Filenames define order
- This file styles every markdown element

> The repo is the deck.
```

</ComponentPreview>

## Installation

`camp init` scaffolds this file for you — install it only to restore
it or to adopt it in an existing deck.

## Usage

There is nothing to call — the runtime picks the file up automatically and
applies it to every slide's markdown. Edit a renderer and the whole deck
restyles:

```tsx title="components/mdx.tsx"
h2: ({ className, ...props }: ComponentProps<"h2">) => (
  <h2
    className={cn("font-serif text-6xl text-(--color-primary)", className)}
    {...props}
  />
),
```

Precedence in the slide component map:

```txt
components/mdx.tsx  >  discovered components  >  defaults
```

That order also means `mdxComponents` can shadow a discovered component by
name — handy for swapping implementations without touching slides.

## API Reference

The file exports a single `mdxComponents` map. Each entry wraps one
intrinsic element, extends its `ComponentProps`, and merges `className`
through `cn` — the same contract as every registry item:

| Key | Renders | Slide-scale default |
| --- | ------- | ------------------- |
| `h1` | `<h1>` | bold, `text-7xl`, tight tracking |
| `h2` | `<h2>` | semibold, `text-5xl`, tight tracking |
| `h3` | `<h3>` | semibold, `text-3xl` |
| `p` | `<p>` | `text-2xl`, relaxed leading |
| `ul` / `ol` | lists | `text-2xl`, spaced, indented |
| `blockquote` | `<blockquote>` | left border, italic, muted |
| `code` | `<code>` | mono, subtle pill background |
| `a` | `<a>` | primary color, underline |

Add any other intrinsic element (`table`, `img`, `hr`, …) by extending the
map — the file is yours.
